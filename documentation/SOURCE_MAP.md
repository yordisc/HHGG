# Mapa de reutilizacion

Este mapa resume que documentos se publican tal cual, cuales solo sirven como referencia y cuales se quedan fuera del espejo.

## Se publica

- [Guia de despliegue](../DEPLOY_RENDER_NEON_AIVEN.md): despliegue y operacion del entorno publico.
- [Guia visual del builder](../VISUAL_BUILDER_GUIDE.md): referencia del editor de certificaciones.
- [Sistema de versionado](../VERSIONING_SYSTEM.md): historial, rollback y comparacion.
- [Troubleshooting](../TROUBLESHOOTING.md): problemas frecuentes y soporte.
- [Guia ZIP de cursos](../COURSE_ZIP_GUIDE.md): paquete ZIP y flujo de exportacion.
- [Banco de preguntas por certificacion](../QUESTION_BANK_CERTIFICATION_GUIDE.md): contrato del banco de preguntas.
- [Prompt de certificacion](../model/CLAUDE_CERTIFICATION_PROMPT.md): prompt de referencia para generar certificaciones.
- [Prompt compacto](../model/CLAUDE_CERTIFICATION_PROMPT_COMPACT.md): version resumida del prompt.
- [Esquema `questions.csv`](../model/questions_csv_schema.md): contrato del CSV dentro del ZIP.

## Apoyo interno

- [README público](./README.md): entrada breve para visitantes.
- [Arbol del portafolio](./PROJECT.md): alcance funcional resumido.
- [Stack tecnico](./TECH.md): piezas principales del sistema.

## Fuera del espejo

- Documentos con detalle interno de operacion, hardening o mantenimiento.
- Notas de seeders, auditorias o fases tecnicas demasiado internas.
- Material pensado para soporte del equipo y no para visitantes externos.

## Regla general

Si un documento ayuda a entender el producto sin exponer detalles internos, se publica. Si solo aporta contexto secundario o historico, se deja como apoyo o se excluye.
